hexo 常见使用方式
Created 2019-11-18| Updated 2019-11-19
Post View:

第一个，题目

hexo g 对更改的内容进行保存在blog中的—_post中 ，之后上传才不会出现上传还是上一次的内容

hexo s 启动server 对本地的hexo 的blog 进行访问 ： localhost:4000

hexo new “name” 创建一个name的md文件 暨自己的一篇文章

hexo d 上传到之前的 github

同时注意 在上传之后翻新所有文件 最好在_post文件中保存README.md

CNAME 方便你的域名进行访问

复制主题方式 git clone github……

或者下载zip保存到相应的位置

在之后的尝试中出现的错误

hexo c

自动清理你的缓存 很多时候会把你的本地index.html 你的博客生成的html给清除了 （我这种喜欢找主题的 很容易就死在这上面

所以在重新 hexo init 一个新的blog 把所有东西搬了一遍 之后

出现了hexo d -g 之后出现 ERROR Deployer not found: git

这时候 使用 npm install --save hexo-deployer-git

进行重新链接你的git

theme设置

下载

我下载的是：butterfly主题，通过github上（ https://github.com/jerryc127/hexo-theme-butterfly/tree/dev ）下载zip包 或者 通过git clone 下载到你的blog文件中。

配置

_config.yel文件中修改theme为 theme: butterfly 之后hexo s 尝试在localhost:4000中打开。观赏效果 同时修改一些本地的数据来更改你的title

author 和其他你不知道的东西

修改butterfly中的config

然后对自己的特效进行修改 然后over 摸索中呢
